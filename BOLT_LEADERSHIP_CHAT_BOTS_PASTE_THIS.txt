## B) غرفة بوت التصميم AI — Chat Room id="owner"

### Room definition
```js
{ id: "owner", name: "بوت التصميم AI", icon: "🎨", category: "private", ownerOnly: true }
```

### Access control
- Only role=owner can enter
- Others get alert: "🎨 غرفة بوت التصميم متاحة للمالك فقط."

### How it works
1. Owner opens room "owner"
2. Owner types ANY normal message (NOT starting with /) in composer
3. Message is intercepted BEFORE sending to Supabase
4. Universal Visual AI Style Engine parses natural language (Arabic + English)
5. Live preview applied to ENTIRE site DOM immediately
6. Bot reply appears in chat + StyleSandboxCard below message

### StyleSandboxCard UI (in chat message list)
Glass card with mini preview samples + 2 buttons:
- ✅ **تطبيق على الكل** → save to owner_settings.universal_style_config + localStorage → all users see it
- ✖ **إلغاء / تعديل** → full rollback to last committed style

### Example owner prompts (Arabic or English)
- "make the site cyberpunk neon"
- "زجاج أكثر ضباب"
- "شريط نور على بطاقات الأعمدة"
- "رجّع الخلفية الافterاضية" → resets wallpaper to /MAN.png
- "مصطلحات" or "help" → vocabulary list of chat regions

### Style regions owner can target
- Site wallpaper / background
- Glass blur, opacity, border radius
- Sidebar column cards (VIP, radio, rooms) — neon chase ring animation
- Chat header strip animation
- Buttons, inputs, message bubbles
- Theme presets: glassmorphic, midnight-luxe, cyberpunk-neon, soft-aurora

### Storage
- Preview: localStorage + DOM only (owner device)
- Commit: Supabase owner_settings.universal_style_config (jsonb)
- Cooldown: 350ms between prompts

### Design Center vs Design Room
- **Design Room (owner)**: type natural language in chat → live preview
- **Leadership → design tab**: upload images, presets, manual controls
- Both use same UniversalStyleConfig schema

## C) CHAT BOTS SYSTEM

### Bot identity
- Author name in messages: "Lamma Bot" or system style
- Message id prefix: `lamma-bot-{timestamp}`
- Bot messages are local UI messages (type: system or special style_sandbox)
- NOT stored in Supabase messages table for bot replies

### Bot 1: LC-Fire (Guard / Protection Bot)
**Scope:** ALL public rooms when bot_enabled=true
**Exempt:** owner and admin never moderated

**Rules (when enabled):**
1. Anti-links: block URLs → warning "🔥 LC-Fire: تم حجب رسالة..."
2. Anti-spam: block identical consecutive message from same author
3. Word wall: block if text contains any banned_words entry

**On violation:**
- Message NOT sent to Supabase
- Bot warning posted to room
- Activity log entry (owner_activity_logs)

**Slash commands (any user, any room):**
```
/help     → command list
/ping     → latency test (queries owner_settings)
/clear    → clear messages on screen (local cosmetic)
/zen      → focus/zen mode
/compact  → compact message view
/guard    → LC-Fire status report (score /100)
/status   → same as /guard
```

**/guard report example:**
```
🔥 تقرير LC-Fire — بوت الحماية:
- حالة البوت: 🟢 نشط
- حجب الروابط: ✅
- منع السبام: ✅
- جدار الكلمات: ✅ (12 كلمة)
- درجة أمان الغرفة: 85/100 🟢 ممتاز
```

**Settings location:** Leadership → guard tab OR admin panel
**Persist:** owner_settings.bot_enabled, bot_rule_anti_links, bot_rule_anti_spam, bot_rule_swear_filter, banned_words

### Bot 2: Games Bot (Games room only)
**Scope:** room id="games" only

**Commands:**
```
/سؤال  /trivia  /quiz  → random trivia question
/تلميح  /hint           → reveal hint
/إجابة [text]           → submit answer (scores tracked)
/نقاط  /score           → leaderboard
/لعبة  /game            → word reverse mini-game
/كلمة                   → word game variant
```

Scores stored in localStorage key: lamma_games_scores

### Bot 3: Design Bot (Owner room only)
See section B above — not slash commands, natural language prompts.

### Bot 4: Store / VIP confirmation bot
When VIP subscription activated → bot message in current room:
"🎉 تم تفعيل اشتراكك في {planName} بنجاح!"

## D) CHAT SYSTEM — Core Architecture

### Rooms (exact IDs from ROOMS_DEF)
```
egypt      مصر 🇪🇬        category: social
arab       كل العرب 🌍     social
youth      لمة شباب وبنات 👫 social
palestine  فلسطين 🇵🇸      social
posts-feed المنشورات 📰    community (social feed UI)
fun        فرفشة 🥳         fun
games      Games 🎮         fun (games bot)
romance    رومانسية 💖      relations
admin      الإدارة والشكاوى 🛡️ social (admin+owner only)
owner      بوت التصميم AI 🎨 private (owner only)
```

Categories sidebar headers: اجتماعي | مجتمع | ترفيه | علاقات | خاصة

### User roles (hierarchy)
```
owner > admin > mod > platinum_vip > vip > user > guest
```

Role badges in member list: BOSS (owner), ADMIN, VIP, PLATINUM

### Message types (Message.type)
```
text | image | video | audio | system | join | leave | gift | youtube | shadow_msg | style_sandbox
```

### Message flow (public rooms)
1. User types in composer → useRoomComposer hook
2. Check: maintenance mode, global mute, room ban, bot moderation
3. If slash command → handleRoomChatCommand (local, no DB)
4. If owner room + owner → tryHandleOwnerStylePrompt (design bot)
5. If games room → handleGameCommand
6. Else → insert to Supabase messages table
7. Realtime subscription broadcasts to all clients in room

### Supabase messages table (key fields)
```
id, room_id, author_nickname, author_color, author_uid (bound server-side),
text, message_type, media_url, gift_icon, youtube_id, reactions (jsonb),
created_at
```

### Realtime
- subscribeToRoomMessages(roomId) — postgres_changes INSERT/UPDATE/DELETE on messages
- Filter by room_id
- Retry logic via subscribeChannelWithRetry

### Composer features
- Text + emoji picker
- Image upload → Supabase Storage bucket "chat-media" (rate limit 3/min)
- Voice recording (if recordingAllowed permission)
- YouTube URL embed (if youtubeAllowed)
- Gift picker (rose, heart, crown, diamond...)
- @mentions → notification badge
- Delete: owner/admin or message author

### Private Messages (PM)
- Separate table: pm_messages
- Realtime per thread
- Mark read (receiver only)
- Spy mode: owner sees all PMs merged in inbox

### Multi-room tabs
- User can open multiple rooms as tabs
- activeRoomId state drives composer + message list
- Each room has own Realtime subscription

### Room switching guards
```
admin room → admin or owner only
owner room → owner only
banned from room → alert block
```

### Notifications
- Sound + browser notification when new message and tab not focused
- PM unread count in header dropdown

### Activity logs (owner)
Types: login, logout, ban, promote, etc.
Stored: owner_activity_logs
Visible in admin/leadership panels
