## 1) راديو لمة (RADIO) — personal listening

### What it is
Each user listens to LIVE internet radio streams on THEIR device only.
NOT synced between users. NOT stored in Supabase.
When owner starts DJ music → radio auto-pauses.

### UI location
Left sidebar, middle section: "راديو لمة" with Radio icon

### UI elements
- Play/Pause button
- Station name + frequency label
- Status: "🔊 على الهواء" or "⏸ متوقف"
- Prev/Next station buttons
- Station list (click to switch)

### Stations (exact — hardcoded RADIO_STATIONS)
```js
[
  {
    id: "quran",
    name: "إذاعة القرآن الكريم 🕋",
    frequency: "مباشر",
    url: "https://backup.qurango.net/radio/tarateel",
    urls: ["https://backup.qurango.net/radio/tarateel", "https://Qurango.net/radio/tarateel"]
  },
  {
    id: "nogoum",
    name: "نجوم FM 📻",
    frequency: "100.6 FM",
    url: "https://audio.nrpstream.com/listen/nogoumfm/radio.mp3"
  }
]
```

### Playback logic
- Use hidden HTMLAudioElement
- playStreamWithFallbacks(audio, urls[]) — try URLs in order, 12s timeout
- toggleRadioPlay: play or pause
- Switching station: pause, clear src, start new station
- If DJ music starts → pause radio automatically

### Permissions
- Owner/admin: always allowed
- Other members: owner can grant `musicRadioAllowed` in غرفة القيادة → صلاحيات الأعضاء
- Default: false for regular members

### NOT radio
✗ Do NOT sync radio between users
✗ Do NOT save radio state to Supabase
✗ Do NOT confuse with DJ broadcast

## 2) DJ Music (Owner Room Broadcast) — synchronized

### What it is
Owner uploads MP3/audio files and BROADCASTS to everyone in the CURRENT chat room.
All listeners hear the SAME song at the SAME position (synced playback).
State saved in Supabase owner_settings.room_dj_map.

### UI location
Left sidebar, bottom section: DJ / موسيقى / music panel

### Owner UI (role=owner only)
- Upload button: accept mp3, wav, ogg, m4a, aac, webm, flac — max 20MB
- Playlist list from dj_library (uploaded tracks)
- Play/Pause → broadcasts to active room
- Prev/Next track in playlist
- Status: "🔊 بث للغرفة" or "⏸ متوقف"
- Shows current track title

### Listener UI (non-owner)
- Toggle button: "🔊 أستمع للـ DJ" / "🔇 مكتوم — اضغط للاستماع"
- Preference saved localStorage key: lamma_dj_listen
- Banner when owner playing: "🎧 المالك يشغّل: {title}"
- When no broadcast: "لا يوجد بث DJ حالياً"
- Listener chooses opt-in — muted by default until they click listen

### Upload flow (owner only)
1. Owner selects audio file(s)
2. Upload to Supabase Storage bucket `chat-media` path: `{uid}/dj/{timestamp}_{uuid}_{filename}`
3. Append to owner_settings.dj_library array:
   `{ id, title, url, fileName, uploadedAt }`
4. Only owner role can upload (server-side role check)

### Broadcast sync flow
When owner presses Play:
1. Build RoomDjState:
```ts
{
  mode: "music",
  trackId: string,
  title: string,
  url: string,           // public URL from storage
  isPlaying: true,
  startedAtMs: number,   // wall-clock ms when track position was 0
  updatedBy: owner nickname,
  updatedAtMs: Date.now()
}
```
2. Save to owner_settings.room_dj_map[activeRoomId] via Supabase UPDATE
3. All clients subscribe to owner_settings changes (or poll room_dj_map)
4. Non-owner clients: applyRoomDjToAudio(hidden audio element, djState)
   - Calculate expected position: (Date.now() - startedAtMs) / 1000
   - Seek audio to that position
   - Drift correction every 3 seconds
   - Only play if isDjListening === true

When owner pauses:
- Remove room entry from room_dj_map (or isPlaying=false)
- All listeners pause

### Per-room broadcast
- DJ state is PER ROOM ID — egypt room DJ ≠ arab room DJ
- Switching rooms → listener hears that room's DJ state (if any)

### Owner pauses radio when starting DJ
- Radio and DJ share left column but only one plays at a time for owner

### Database fields
```
owner_settings.dj_library: OwnerMusicTrack[]
owner_settings.room_dj_map: Record<roomId, RoomDjState>
```

## 3) يوتيوب / فيdeo (YOUTUBE & VIDEO)

### What it is
Share YouTube links or direct video URLs in chat messages.
YouTube renders as embedded iframe player in message bubble.

### How to send (2 ways)

**Way A — Attachment menu:**
- User clicks attachment → "فيديو / يوتيوب"
- Prompt: "🎥 أدخل رابط يوتيوب أو فيdeo مباشر (MP4, WEBM):"
- Sends message type="video" with mediaUrl

**Way B — Paste in text:**
- User types message containing youtube.com or youtu.be URL
- Composer detects URL before send
- Same permission check applies

### YouTube detection
```js
getYoutubeId(url):
  regex: youtu.be/ | v/ | embed/ | watch?v=
  returns 11-char video ID or null
```

### Message rendering
If getYoutubeId(mediaUrl) exists:
```html
<iframe src="https://www.youtube.com/embed/{videoId}" />
```
Else if direct MP4/WEBM URL → HTML5 video element

### Permissions
- Owner/admin: always allowed
- Others: `youtubeAllowed` in owner_member_permissions (default false)
- Block message + alert if not allowed:
  "🎥 مشاركة يوتيوب/فيديو غير مفعّلة — غرفة القيادة → صلاحيات الأعضاء"

### Supabase message fields
```
message_type: "video"
media_url: full URL
youtube_id: extracted ID (nullable)
```

### Anti-link bot interaction
- LC-Fire anti-links blocks external URLs in TEXT messages
- YouTube via attachment menu is intentional media — separate path
- Owner/admin exempt from link blocking

### PM support
- Same YouTube embed in private messages
- Same permission checks

## 4) الصور (IMAGES)

### What it is
Upload image files or paste image URLs in public chat and PM.

### How to send (2 ways)

**Way A — File upload:**
- Attachment menu → "صورة"
- Opens file picker (image/* only)
- Upload then send

**Way B — Image URL:**
- Attachment → "رابط صورة"
- Prompt: "🔗 أدخل رابط صورة (jpg/png/webp/gif)"
- Sends without upload

### Upload specs
- Bucket: `chat-media`
- Path: `{uid}/rooms/{roomId}/{timestamp}_{uuid}_{filename}`
- Max size: 5MB
- Types: image/* only
- Requires registered Supabase account (NOT guest anonymous for upload)
- Rate limit: max 3 media uploads per 60 seconds per user

### Permissions (3 layers)
1. Owner/admin: always
2. Member has `imagesAllowed` from owner (غرفة القيادة)
3. OR if owner enabled `vip_only_images` toggle → VIP/platinum/mod roles can send

Default: regular users CANNOT send images until owner grants.

Block alert:
"📸 ميزة الصور غير مفعّلة — غرفة القيادة → صلاحيات الأعضاء"

### Message rendering
- type="image"
- Show <img> with mediaUrl, click to expand/lightbox
- Lazy load, rounded corners in glass bubble

### Owner global toggle
Leadership → quick → "🖼️ الصور للـ VIP فقط" (vip_only_images in owner_settings)

### PM images
- Upload to `{uid}/pm/{targetNickname}/...`
- Same permission rules

## 5) تسجيل الصوت / رسائل صوتية (VOICE MESSAGES)

### What it is
WhatsApp-style voice notes in chat — NOT live calls.
Record → upload → play inline bubble.

### UI flow
1. User clicks mic icon in composer attachment menu
2. VoiceRecorderBar appears above composer:
   - Waveform animation
   - Timer MM:SS
   - Cancel button (discard)
   - Send button (upload + post)
3. After send: VoiceNoteBubble in message list with play/pause + progress bar

### Recording tech
- navigator.mediaDevices.getUserMedia({ audio: echoCancellation, noiseSuppression })
- MediaRecorder with mime: audio/webm;codecs=opus (fallback mp4/ogg)
- Collect chunks every 250ms
- Min duration: 1 second

### Upload
- Bucket: chat-media
- Path: `{uid}/voice/{roomId}/{timestamp}_{uuid}.webm`
- Message type="audio", mediaUrl=public URL

### Permissions
- Owner: always
- Others: `recordingAllowed` from owner (default false)
- Must be registered Supabase user (not guest)
- Blocked if owner enabled `global_mic_mute` (except owner/admin)

Alerts:
- "⚠️ ميزة الرسائل الصوتية غير مفعلة من قبل المالك"
- "🎙️ الميكروفونات مقفلة من قبل المالك"

### VoiceNoteBubble UI
- RTL glass card ~200-320px wide
- Play/pause circle button (green)
- 18-bar waveform visualization
- Label: "🎤 رسالة صوتية"
- Duration display + seek progress bar
- Hidden <audio playsInline preload="metadata">

### Rate limit
Same as images: 3 media uploads per minute

### PM voice
- Supported in private messages with same recorder

## 6) المكالمات صوت + فيdeo (WebRTC CALLS)

### What it is
Real-time peer-to-peer audio/video calls between 2 users.
ONLY from Private Messages (PM) panel — NOT in public rooms.

### UI entry points
PM header (when chatting with someone):
- 📞 Phone button → audio call
- 📹 Video button → video call

### Permissions
| Type | Permission field | Who default |
|------|-----------------|-------------|
| Audio call | callsAllowed | owner always; others need grant |
| Video call | videoCallsAllowed | owner always; others need grant |

Guests (role=guest) BLOCKED entirely:
"رتبة زائر غير مصرح لها بإجراء المكالمات — سجّل حساب"

Grant from: غرفة القيادة → صلاحيات الأعضاء

### WebRTC architecture
```
Caller                          Supabase                    Receiver
  │                          call_signals                    │
  ├── INSERT offer ────────────────────────────────────────►│ Realtime subscribe
  ├── INSERT ice candidates ───────────────────────────────►│
  │◄──────────────── INSERT answer ─────────────────────────┤
  │◄──────────────── ICE candidates ────────────────────────┤
  └── RTCPeerConnection ←──────── P2P media ──────────────►│
```

### Signaling table: call_signals
```
call_id, from_uid, from_nickname, to_uid, to_nickname,
call_type: "audio" | "video",
signal_type: ring | offer | answer | ice | accept | reject | hangup | server-switch,
payload: jsonb (sdp, candidate, etc.)
```

### ICE servers (dual bundle failover)
Primary: Google STUN + public TURN (openrelay.metered.ca)
Fallback: Cloudflare STUN + alternate TURN
Optional env: VITE_TURN_URL, VITE_TURN_USERNAME, VITE_TURN_CREDENTIAL

### Call states UI
1. **Outgoing:** "connecting" → "ringing" (45s timeout)
2. **Incoming popup:** caller name, accept/reject buttons, audio or video icon
3. **Connected:** duration timer, local preview (video), remote stream
4. **Failed/reconnecting:** auto ICE bundle switch after 4s
5. **Ended:** cleanup streams

### Call overlay
Full-screen or floating widget:
- Local video (picture-in-picture for video calls)
- Remote video/audio element
- Mute, camera toggle, end call button
- Label: "مكالمة فيديو WebRTC" or "مكالمة صوتية WebRTC"

### Resolve peer UID
- Find online member by nickname in chatMembers list
- Must have valid UUID auth uid — cannot call offline users

### Spy mode
- Owner spy PM threads: call buttons HIDDEN (cannot call spied conversations)
